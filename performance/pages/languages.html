Is Java that much slower than C99/C++?

Yes and no.

  • Long running CPU-bound tasks: not really/not by a lot (surprise..!)
  • JIT is slower but can allow for platform specific optimizations during runtime

Overall

  • I/O/arithmetic ops/exception -> same ballpark
  • Strings are slower -> immutable UTF-16 encoded objects
  • Trigonometric funs -> due to tight spec might != opcodes
  • Java uses a lot of memory -> object model overhead
  • Harder to implement lower level data-structures in Java - Make sun.misc.Unsafe Great Again!
  • GC requires expertise in tightly-bounded applications (finance, HFT, real-time systems)
  • JVM actually has lower overhead for allocation of memory than malloc/free (it doesn't really allocate to be fair..)

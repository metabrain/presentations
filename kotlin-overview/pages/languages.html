JVM Languages overview

Focus for today

  • Java (surprise..!)
  • Kotlin
  • Groovy

Honorable mentions

  • Scala
  • Closure
  • Ceylon

Scala has too many features to list here and the problem space can be quite different - plus it's a very academic language.

Closure does not improve on Java-esque syntax but rather replaces it with a completelly distinct one.

Ceylon is not as used/matured as the other and quite honestly, I do not know much of it... Other than it has Union Types which is awesome!

Why focus on Kotlin

  • Small learning curve for Java devs compared to other JVM languages (Scala/Closure)
  • A lot less verbose than Java - an order magnitude such as the jump between Java 8 and 5/6
  • Share same JVM bytecode ecosystem - all Java libraries can be used in Kotlin, which some nuances
  • Some AdCloud services as off late have been "accidentaly" written in Kotlin, so I have some experience using it. (yes, it is in production)
